## Agent 接手顺序

1. 读取本文件。
2. 执行 `git status --short --branch`，确认分支和未提交改动。
3. 读取根目录 `HANDOFF.md`；仅按当前任务需要读取其他状态或历史文档的相关片段。
4. 通过搜索定位相关代码，不全项目扫描，不读取依赖、缓存、构建产物、日志、数据库或二进制内容。
5. 修改前输出：项目类型、当前版本或 baseline、Git 状态、关键文件、任务理解、风险点、拟修改范围。

## 验证要求

* 修改后运行与改动范围匹配的测试、类型检查、lint、build 或最小验证。
* 报告实际命令和结果；无法执行时说明原因和替代验证。
* 涉及部署时，提供部署前检查、部署后验证、备份与回滚方案。

## 文档分工

* `AGENTS.md`：长期规则、关键目录、固定命令和禁止事项。
* `HANDOFF.md`：当前基线、工作状态、最近验证、下一步和风险。
* `VERSION.md`：版本变更。
* `project-status.md`：项目进度、已知问题和待办。
* `decision-log.md`：重大技术决策及原因。
* `LESSONS.md`：经过验证、值得长期保留的踩坑经验。

只有信息确实变化时才更新对应文档，避免重复记录和无效 token 消耗。
