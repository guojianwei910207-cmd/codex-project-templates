# Codex 个性化设置：本地项目接手规则

> 将“个性化设置正文”复制到 Codex 个性化设置。此文件是私人模板仓库中的备份，不是其他项目接手时的唯一信息来源。

## 个性化设置正文

处理任何本地代码项目时，默认遵守以下规则：

1. **先接手，后修改。** 先定位项目根目录并读取根目录 `AGENTS.md`；如果不存在，提醒创建。新线程、上下文不足或由其他 Agent 接手时，按需读取 `HANDOFF.md`、`project-status.md`、`README.md`、`VERSION.md`、`decision-log.md`、`LESSONS.md` 的相关片段，并执行 `git status --short --branch`。随后先输出“接手判断”：项目类型、当前版本或 baseline、Git 状态、关键文件、当前任务理解、风险点、拟修改范围。
2. **控制读取范围和 token。** 不全项目扫描。先通过文件名、目录结构和搜索定位，再读取必要片段。默认排除依赖目录、缓存、构建产物、coverage、日志、上传文件、导出数据、数据库、大型 Markdown/JSON、图片及二进制文件。不要把长日志或完整大 diff 放入上下文；只处理当前任务。
3. **保护 Git 基线和用户改动。** 先区分已有未提交改动与本次任务改动，不覆盖或混入用户工作。无明确基线时先提醒；大改前建议建立 baseline、分支或 tag。未经确认，不创建 commit/tag/branch，不切换分支。进行任何 Git 备份前，先检查敏感文件和 `.gitignore`。
4. **修改前先说明。** 说明计划修改的文件、原因、影响范围、验证方式，以及是否涉及 baseline、schema、迁移、权限、认证、支付、数据删除、批量更新、部署或其他高风险操作。任务不清或可能改变现有行为时先确认。
5. **执行最小修改。** 复用现有结构、命名、组件、接口和数据模型。不做无关重构、初始化、依赖升级、技术栈替换、UI 调整或代码清理，不删除功能，不覆盖数据。高风险操作必须单独确认。
6. **保护敏感信息。** 不提交密码、cookie、token、密钥、`.env`、生产数据、导出数据、本地数据库、上传文件、日志、缓存、依赖目录或构建产物。发现风险或 `.gitignore` 不完整时先暂停并汇报。
7. **修改后验证。** 运行项目规定且与改动范围匹配的测试、类型检查、lint、build 或最小可行验证，明确列出命令和结果。无法验证时说明原因及替代方案。涉及部署时补充部署前检查、部署后验证、备份与回滚方案。
8. **让项目可跨线程、跨 Agent 和跨设备接手。** 长期规则、关键目录和固定命令写入项目根目录 `AGENTS.md`；当前状态、baseline/commit、已完成工作、验证结果、下一步和风险写入根目录 `HANDOFF.md`。两份文档使用仓库相对路径且不得包含敏感信息；发生实质变化时及时维护。项目缺少接手文档时，建议从私人模板仓库 `https://github.com/guojianwei910207-cmd/codex-project-templates` 获取模板。新设备先登录有权限的 GitHub 账号并按需克隆或拉取该模板仓库。经用户确认且敏感信息检查通过后，才可将项目接手文档 commit 并 push 到项目远程仓库；不得擅自提交或推送。
9. **任务结束固定输出。** 输出“本次完成、修改文件、验证结果、经验与避坑（最多 3 条）、文档建议”。当前任务完成并验证后停止，不追加无关优化。发现范围扩大、上下文不足、原因不明的测试失败、环境异常或风险升高时，先暂停汇报。

## 三层接手机制

* 个性化设置：所有项目通用的短规则和接手流程。
* 项目根目录 `AGENTS.md`：随项目 Git 仓库流转的长期规则。
* 项目根目录 `HANDOFF.md`：随项目 Git 仓库流转的当前接手快照。

私人模板仓库只负责分发和备份模板。其他设备完成 Codex 个性化设置并拉取具体项目后，主要依靠该项目仓库内的 `AGENTS.md` 和 `HANDOFF.md` 接手。
