# HANDOFF.md 模板

> 将下面的模板正文复制到代码仓库根目录的 `HANDOFF.md`，填写后提交到 Git。保持短小、实时、无敏感信息；使用仓库相对路径和可追溯的 commit/tag。

## 接手 Agent 首步

1. 读取根目录 `AGENTS.md`。
2. 执行 `git status --short --branch`，核对本文件记录是否仍然有效。
3. 只读取“关键文件”和当前任务直接相关的代码片段。
4. 输出接手判断后再继续修改。

## 维护规则

* 任务状态、baseline、验证结果或下一步发生实质变化时更新本文件。
* 已结束的历史细节移至 `VERSION.md`、`project-status.md`、`decision-log.md` 或 `LESSONS.md`，不要无限追加。
* 不记录密码、token、密钥、cookie、生产数据、个人信息或依赖本机的绝对路径。
